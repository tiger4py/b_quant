# 策略评估分析 Skill

对任意回测策略做深度诊断，不跑新回测，只基于已有 archive JSON 做分析。

## 触发条件

用户说"分析策略"、"评估策略"、"诊断策略"、"review strategy"、"这份回测报告看一下"时触发。

## 输入

用户需指定策略 ID（如 `trend_following`），skill 自动找最新的 archive JSON。

## 分析流程（6 个维度）

### 1. 概览

从 `summary` 提取：总收益、最大回撤、胜率、交易次数、平均盈亏、盈亏比。

### 2. 卖出原因拆解

```python
# 按 sell_reason 字段分组，计算每组的：笔数、占比、平均盈亏、胜率、累计盈亏
# 定位"杀手"——哪类卖出原因在吞噬利润
```

输出：每个卖出原因的行数、平均盈亏、胜率、累计贡献。标注最大亏损源和最大利润源。

### 3. 盈亏分布

```python
# 盈亏区间分桶：[-100,-12), [-12,-5), [-5,0), [0,5), [5,15), [15,25), [25,100]
# 按年统计收益
# 最大/最小单笔盈亏、中位数
```

输出：盈亏区间分布表 + 按年收益表。看收益是否集中在少数交易。

### 4. 买入信号区分力

```python
# 从 buy_reason 里提取：5日涨幅、20日涨幅、量比、涨跌量比、连涨天数
# 按卖出结果分组（止盈 vs 到期 vs 止损），对比各指标的均值和范围
```

输出：止盈组 vs 到期组 vs 止损组的买入信号特征对比表。

**关键判断**：如果三组的买入信号几乎一模一样 → 买入条件没有区分力，策略靠运气。

### 5. 卖出后追踪（最重要）

```python
# 对每笔交易的卖出日期，从数据库查后续 5/10/22/44/66 个交易日的收盘价
# 计算相对卖出价的涨跌%
# 按卖出原因分组统计中位数和涨跌比
```

输出：每个卖出原因在 1周/2周/1月/2月/3月后的中位数涨跌。

**关键判断**：
- 中位数全线为负 → 卖出时机对
- 中位数为正 → 卖早了
- 止盈组卖后继续跌 → 止盈点合理
- 到期组卖后继续跌 → 不用延长

### 6. 假设场景模拟

基于已查到的后续价格数据，模拟"如果参数变了会怎样"：

```python
# 场景 A: 止盈阈值从 25% 改到 20% → 哪些交易会受影响？
# 场景 B: 持仓天数从 14 改到 22 → 到期交易的盈亏如何变化？
# 场景 C: 止损放宽到 -12% → 止损组有多少能回本？
```

输出：每个场景的模拟收益变化，以及是否推荐。

## 不允许做的事

- **不跑新的全市场回测**（太慢，等用户决定再跑）
- **不修改策略代码**（只分析，不修改）
- **不自动应用"优化建议"**（先给用户看数据，用户决定）

## 输出格式

用表格 + 简短结论，最后给一个"操作建议"区块（≤3 条建议）。
